# MLP Nonlinearity Compensation Pipeline — Complete SOP

## Overview

This document records the end-to-end workflow, code architecture, and pitfalls for the MLP-based fiber nonlinearity compensation (NLC) pipeline. The pipeline spans three phases: (B1–B2) SSFM fiber transmission simulation, (B3) lightweight MLP compensator training, and (B4) performance evaluation with BER/Q-factor/complexity analysis. The target system is a single-channel 16-QAM, 32 GBaud transmission over 10×80 km standard single-mode fiber (SSMF).

**Reference paper:** MT-NN Simplified Architecture (Optics Express, 2025) — complexity-aware lightweight MLP for fiber nonlinearity compensation.

---

## 1. Core Physics & Key Formulae

### 1.1 Nonlinear Schrodinger Equation (NLSE)

$$\frac{\partial A}{\partial z} = -\frac{\alpha}{2}A - j\frac{\beta_2}{2}\frac{\partial^2 A}{\partial t^2} + j\gamma|A|^2 A$$

| Symbol | Meaning | Value Used |
|--------|---------|-------------|
| $\alpha$ | Field attenuation coefficient | $4.605 \times 10^{-5}$ m⁻¹ (0.2 dB/km) |
| $\beta_2$ | GVD parameter | $-2.17 \times 10^{-26}$ s²/m (D=17 ps/nm/km @1550nm) |
| $\gamma$ | Kerr nonlinearity coefficient | $1.3 \times 10^{-3}$ W⁻¹m⁻¹ (1.3 /W/km) |

### 1.2 Symmetric Split-Step Fourier Method (SSFM)

Per-step operator sequence: ½N → D → ½N → attenuation

$$A(z+dz) \approx \exp\left(j\gamma|A|^2 \frac{dz}{2}\right) \cdot \mathcal{F}^{-1}\left\{\exp\left(-j\frac{\beta_2}{2}\omega^2 dz\right) \mathcal{F}\{A\}\right\} \cdot \exp\left(j\gamma|A|^2 \frac{dz}{2}\right) \cdot \exp\left(-\frac{\alpha}{2}dz\right)$$

Key: the dispersion operator D applies the **full** $dz$ step (not $dz/2$), because the ½-splitting of nonlinearity is what makes it "symmetric."

### 1.3 EDFA Amplification & ASE Noise

$$G = e^{\alpha L_{span}} \quad\text{(linear power gain)}$$

$$\sigma_{ASE}^2 = N_{sp} \cdot h \cdot \nu \cdot (G - 1) \cdot \frac{1}{dt}$$

$$A_{out} = A_{in} \cdot \sqrt{G} + n_{ASE}, \quad n_{ASE} \sim \mathcal{CN}(0, \sigma_{ASE}^2)$$

Critical ordering: **amplify first, then add noise**. The signal arriving at the EDFA has been attenuated by `exp(-α·L_span/2)` (field). The amplifier must restore it before noise injection.

### 1.4 Electronic Dispersion Compensation (EDC)

$$H_{EDC}(\omega) = \exp\left(+j\frac{\beta_2}{2}\omega^2 L_{total}\right), \quad L_{total} = N_{spans} \times L_{span}$$

EDC is a linear all-pass filter. It perfectly inverts chromatic dispersion but is blind to the intensity-dependent Kerr phase shift. After EDC, the residual distortion is dominated by SPM-induced nonlinear phase noise.

### 1.5 16-QAM Gray Mapping

Constellation: $\{\pm1 \pm 1j, \pm1 \pm 3j, \pm3 \pm 1j, \pm3 \pm 3j\} / \sqrt{10}$

The $1/\sqrt{10}$ normalization ensures $E[|s|^2] = 1$. Gray code table: `[0, 1, 3, 2, 4, 5, 7, 6, 12, 13, 15, 14, 8, 9, 11, 10]`.

Mapping: integer index (0..15) → Gray encode → grid position `(row, col)` → `i = 2*row-3, q = 2*col-3`.

### 1.6 Performance Metrics

| Metric | Formula | Notes |
|--------|---------|-------|
| EVM | $\sqrt{\frac{1}{N}\sum|s_{rx} - s_{tx}|^2}$ | RMS error, normalized to unit-power TX |
| Q-factor | $20\log_{10}(1/\text{EVM})$ | dB |
| BER (theoretical) | $\frac{3}{4}\text{erfc}\left(\sqrt{\frac{\text{SNR}_{sym}}{10}}\right)$ | 16-QAM Gray-coded approximation |
| BER (counting) | Direct hard-decision Gray demapping | Conservative; limited by test set size |

## 3. Pitfalls & Fixes

### 3.1 SSFM Simulation Pitfalls (B1–B2)

See `SSFM_SOP.md` Section 3 for the following sub-pitfalls:
- **Pit 1:** Unit mismatch — ALPHA_LIN per-km vs per-meter → overflow
- **Pit 2:** SSFM dispersion step — used D(dz/2) instead of D(dz)
- **Pit 3:** Field attenuation — used α instead of α/2
- **Pit 4:** RRC filter NaN at t=0 and t=±1/(4β)
- **Pit 5:** EDFA missing signal re-amplification before noise injection
- **Pit 6:** Symbol timing misalignment from convolution mode mismatch
- **Pit 7:** V&V phase estimator destroying noisy 16-QAM constellation
- **Pit 8:** UnicodeEncodeError in Windows GBK terminal
- **Pit 9:** pip proxy blocking PyPI mirrors

### 3.2 MLP Training Pitfalls (B3)

#### Pit 10: Feature/Target Alignment in Sliding Window

**Symptom:** After building features with memory M, the center symbol index in the feature vector and the target extraction must align exactly.

**Root cause:** `build_features()` trims M symbols from each end. `build_target()` must trim the same M symbols. If one trims differently, feature row k no longer corresponds to target row k.

**Fix:** Both functions use identical loop bounds:
```python
for k in range(memory, n - memory):  # identical in both functions
```
Feature row i corresponds to symbol index `i + memory` in the original sequence.

#### Pit 11: Power Normalization Drift Between Dataset and Evaluation

**Symptom:** MLP-trained output symbols show residual scaling error in BER counting.

**Root cause:** The EDC output symbols are power-normalized in `ssfm_simulator.py`:
```python
rx_symbols /= np.sqrt(np.mean(np.abs(rx_symbols) ** 2))
```
This normalization is approximate (sample mean, not ensemble). The TX symbols are exactly unit power by construction. When the MLP learns the mapping rx → tx, any slight mis-normalization in rx becomes a learned bias.

**Fix:** Accept this as inherent. The MLP naturally learns to compensate for minor scaling deviations as part of the MSE minimization. Verify that EVM between rx_edc and tx is measured on consistently normalized data.

#### Pit 12: Model Checkpoint Loading with `weights_only=False`

**Symptom:** `torch.load()` with default `weights_only=True` (PyTorch >=2.6) fails when loading the full checkpoint dict (models + config + results).

**Root cause:** The checkpoint contains Python dicts and lists alongside tensor data. `weights_only=True` only allows tensor primitives.

**Fix:**
```python
checkpoint = torch.load(MODEL_PATH, map_location=DEVICE, weights_only=False)
```
This is required because we save metadata (config, results) alongside model weights.

### 3.3 Performance Evaluation Pitfalls (B4)

#### Pit 13: Direct BER Counting Floor Limited by Test Set Size

**Symptom:** MLP counting BER at 0/+2/+4 dBm shows 1×10⁻⁸⁰ (zero errors detected), but the theoretical BER from EVM is far lower (10⁻²⁰⁰+). Both are "essentially zero," but the counting method cannot resolve BER below ~1/N_bits.

**Root cause:** Test set has only ~2,453 symbols × 4 bits ≈ 9,800 bits per power level (15% of 16,384). Minimum measurable BER ≈ 1×10⁻⁴. The theoretical estimate has no such floor, but underflows erfc to 0 at high SNR.

**Fix:** Use both methods and report transparently:
- **Theoretical BER** — meaningful for Gaussian-noise-dominated regimes (low power)
- **Counting BER** — provides a conservative upper bound; "zero errors in N bits" is reported as BER < 1/N
- For publication, increase N_symbols (e.g., 2¹⁸ = 262,144) to push the counting floor lower

#### Pit 14: Gray Code Demapping Double-Lookup Coincidence

**Symptom:** The Gray demapping code appeared to work correctly despite an apparent off-by-one error.

**Root cause:** The `symbol_to_bits()` function computes `gray[gray_idx]` where `gray_idx` is the grid position (= Gray code value itself). This applies the Gray transform *twice*. For the 4-bit binary reflected Gray code, every element lies in a cycle of length 1 or 2 under the Gray permutation — i.e., `gray[gray[x]] = x` for all x. The double-lookup is **accidentally an identity**.

**Fix:** The code is correct by coincidence. The proper implementation would use the inverse Gray mapping directly:
```python
# Explicit inverse (correct but equivalent to current coincidental version):
inv_gray = {g: idx for idx, g in enumerate(gray_code_4bit())}
original_idx = inv_gray[grid_position]  # grid_position IS the Gray code
```
Both approaches give identical results. The current code is safe to keep.

#### Pit 15: DBP Complexity Underestimation (Signal Length vs Symbol Count)

**Symptom:** Initial B4 script estimated DBP at only 3,500 ops/symbol, making MLP look comparable rather than dramatically simpler.

**Root cause:** FFT complexity was computed using `5 × N_symbols × log2(N_symbols)` instead of `5 × N_samples × log2(N_samples)`. With SPS=4, the actual signal length is 65,536 samples (not 16,384). Each FFT operates on the full upsampled signal.

**Fix:**
```python
# WRONG:
FFT_OPS = 5 * 16384 * np.log2(16384)       # ~1.1M ops/FFT
# CORRECT:
N_SAMPLES = N_SYMBOLS * SPS  # 65536
FFT_OPS = 5 * N_SAMPLES * np.log2(N_SAMPLES)  # ~5.2M ops/FFT
```
After correction: Standard DBP ≈ 3.44×10⁶ ops/symbol, MLP ≈ 3.23×10³ ops/symbol. MLP is ~1,064× simpler.

#### Pit 16: Inconsistent Dataset Split Between Training and Evaluation

**Symptom:** If B4 evaluation uses a different random split than B3 training, the test set differs and results are not comparable.

**Root cause:** Both scripts must use identical `RandomState(RANDOM_SEED).permutation(n)` with the same seed.

**Fix:** Hard-code `RANDOM_SEED = 42` in both scripts. Ensure the split ratios (70/15/15) are identical. The `eval_performance_b4.py` script explicitly replicates the training split logic.

## 4. Key Design Decisions

1. **Per-power-level independent models vs. single unified model** — Chose independent models. Rationale: each power level has different nonlinear-to-ASE ratios. A single model would need power level as an additional input feature. Independent models are simpler and directly show the power-vs-gain relationship.

2. **Memory taps M=2 (5-symbol window)** — Based on the MT-NN simplified architecture. At D=17 ps/nm/km over 800 km, the chromatic dispersion spreads a symbol over ~4-5 symbol periods. M=2 is the minimum that captures first-order ISI while keeping input dimension small (10 features).

3. **Hidden layer sizes [64, 32, 16]** — Decreasing pyramid structure. First layer (64 neurons from 10 inputs) provides an overcomplete representation. Subsequent contraction forces the network to learn a compressed nonlinearity manifold. Total 3,346 parameters is deliberately compact.

4. **ReLU activation over Tanh** — ReLU avoids vanishing gradients in deeper stacks, trains faster, and has lower inference cost (single compare vs. transcendental). The IQ regression task has no saturation concerns that would favor Tanh.

5. **MSE loss (not BER-aware loss)** — MSE on I/Q coordinates is convex and well-behaved. Direct BER minimization is non-differentiable. The Gaussian-noise assumption implicit in MSE aligns with the ASE-dominated regime.

6. **No batch normalization** — With only 3,346 parameters and ReLU activations, the network is shallow enough to train without batchnorm. Adding it would complicate inference (running mean/variance tracking).

7. **"Same"-mode convolution at both TX and RX** — Produces zero net timing delay when using identical filters at both ends. Verified via back-to-back EVM < 0.002.

8. **Adaptive dz in SSFM** — `dz_actual = L_span / floor(L_span / dz_nominal)` ensures exactly integer steps per span, preventing accumulation of fractional-step truncation errors over 10 spans.

## 6. Reproducibility Notes

1. **Fixed random seeds:** `RANDOM_SEED = 42` in both `torch.manual_seed()` and `np.random.seed()`, plus `np.random.RandomState(42)` for data splitting.
2. **Single symbol generation:** `generate_16qam_symbols()` is called once; the same 16,384 symbols are reused across all 5 power levels for fair comparison.
3. **Deterministic SSFM:** The SSFM itself is deterministic (no randomness in propagation). Randomness enters only through ASE noise and symbol generation.
4. **Dataset caching:** `fiber_dataset.npz` is the single source of truth. Regenerate it only when changing physical parameters.
5. **GPU determinism:** Not enforced (training on CPU or GPU may produce slightly different weight values due to floating-point non-associativity). Results are qualitatively identical.
